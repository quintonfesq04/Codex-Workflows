# Carousel Example Intake

Use this checklist when the user sends screenshots or finished carousel examples for a specific format.

## Ask For

- Cover example.
- Standard info slide example.
- Stat-heavy slide example, if different.
- Meme slide example, if the format uses memes.
- One rejected example, if the user has a clear "not this" reference.

## Capture

- Canvas and safe area: confirm 1080x1350 and visible margins.
- Cover crop: where the face/body sits, how much subject fills the top half, logo side.
- Cover text: line count, relative size difference between lines, gap, bottom margin, color.
- Gradient: color, opacity, where it starts, where it becomes solid, where it fades out.
- Info slides: white copy band height, text size, line count, image crop, stat location.
- Stat treatment: fill color, stroke treatment, size, position, and contrast.
- Meme slides: caption band height, caption size, meme crop, whether any extra branding appears.
- Format-specific details: number badges, player labels, source screenshots, tables, or quote blocks.

## Do Not Capture

- Private local file paths.
- Downloaded images.
- Paid asset files.
- Meme-bank originals.
- Fonts.
- Credentials or browser/session artifacts.

## Output Note Format

Add notes to the matching format file using this shape:

```text
## Approved Look Notes

- Cover:
- Info slide:
- Stat line:
- Meme:
- Logo:
- Color/gradient:
- Things to avoid:
```

Keep the notes specific enough that Codex can reproduce the look without needing the original chat.
